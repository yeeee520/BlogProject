## 1. Git 版本控制最简工作流

### 1.1 为什么要用 Git？（替代复制备份）

| 方式 | 问题 |
|------|------|
| 复制 `project-final/`、`project-final-2/` | 文件冗余、不知道改了啥、合并冲突时崩溃 |
| Git | 记录每一次改动、可回退到任意版本、可查看"谁在什么时候改了哪一行" |

### 1.2 最小初始化流程

```bash
# 1. 在项目根目录初始化
git init

# 2. 配置用户信息（仅首次）
git config user.name "your-name"
git config user.email "your-email@example.com"

# 3. 添加文件到暂存区
git add .

# 4. 提交到本地仓库
git commit -m "feat: 初始化 SpringBoot 项目"

# 5. 后续修改后重复 3-4 两步
git add .
git commit -m "feat: 完成用户登录接口"
```

> **黄金法则**：完成一个**小功能**就 `add` + `commit` 一次，不要攒三天的代码才提交。

## 2. 后端技术选型清单

| 组件 | 作用 | 为什么选它 |
|------|------|-----------|
| **Spring Boot 3.x** | 应用程序框架，自动配置、嵌入式 Tomcat | 业界标准，开箱即用，新手友好 |
| **MyBatis-Plus** | ORM 框架，操作数据库 | 比 MyBatis 原生省掉大量 XML，自动 CRUD |
| **MySQL 8.x** | 关系型数据库 | 免费、稳定、生态丰富 |
| **Spring Security** | 认证与授权框架 | 拦截未登录请求、管理角色 |
| **JWT (jjwt)** | 无状态 Token，用户登录后发放 | 无需服务端 Session，适合前后端分离 |
| **Knife4j** | API 文档自动生成 + 在线调试 UI | 基于 Swagger，界面更美观、调试更方便 |
| **Lombok** | 消除 Getter/Setter/构造器等样板代码 | 少写 40% 的代码 |
| **MapStruct** | 实体与 DTO 之间的对象映射 | 编译期生成代码，比 BeanUtils 快 10 倍 |
| **Hibernate Validator** | 参数校验注解 | `@NotBlank` 一行搞定校验，无需手写 if |
| **Redis** (可选推荐) | 缓存、点赞去重、Token 黑名单 | 提升性能，但初期可以不用 |

## 3. 数据库表设计

### 3.1 ER 概览

```
user ──1:N──> article ──1:N──> comment
user ──1:N──> like_record ──N:1──> article
```

### 3.2 完整建表 SQL

```sql
-- ========================================
-- 1. 用户表
-- ========================================
CREATE TABLE `user` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT "主键ID",
    `username`    VARCHAR(32)  NOT NULL                COMMENT "用户名（唯一）",
    `password`    VARCHAR(128) NOT NULL                COMMENT "BCrypt 加密后的密码",
    `nickname`    VARCHAR(64)  DEFAULT NULL            COMMENT "昵称",
    `avatar`      VARCHAR(256) DEFAULT NULL            COMMENT "头像 URL",
    `email`       VARCHAR(128) DEFAULT NULL            COMMENT "邮箱",
    `bio`         VARCHAR(256) DEFAULT NULL            COMMENT "个人简介",
    `role`        VARCHAR(16)  NOT NULL DEFAULT "user" COMMENT "角色：user/admin",
    `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT="用户表";

-- ========================================
-- 2. 文章表
-- ========================================
CREATE TABLE `article` (
    `id`            BIGINT       NOT NULL AUTO_INCREMENT COMMENT "主键ID",
    `user_id`       BIGINT       NOT NULL                COMMENT "作者ID",
    `title`         VARCHAR(128) NOT NULL                COMMENT "文章标题",
    `summary`       VARCHAR(256) DEFAULT NULL            COMMENT "文章摘要",
    `content`       LONGTEXT     NOT NULL                COMMENT "文章内容（Markdown）",
    `cover_image`   VARCHAR(256) DEFAULT NULL            COMMENT "封面图",
    `category`      VARCHAR(32)  DEFAULT NULL            COMMENT "分类",
    `tags`          VARCHAR(256) DEFAULT NULL            COMMENT "标签，逗号分隔",
    `status`        TINYINT      NOT NULL DEFAULT 0      COMMENT "0=草稿, 1=已发布",
    `view_count`    INT          NOT NULL DEFAULT 0      COMMENT "浏览量",
    `like_count`    INT          NOT NULL DEFAULT 0      COMMENT "点赞数",
    `comment_count` INT          NOT NULL DEFAULT 0      COMMENT "评论数",
    `create_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_category` (`category`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT="文章表";

-- ========================================
-- 3. 评论表
-- ========================================
CREATE TABLE `comment` (
    `id`          BIGINT   NOT NULL AUTO_INCREMENT COMMENT "主键ID",
    `user_id`     BIGINT   NOT NULL                COMMENT "评论者ID",
    `article_id`  BIGINT   NOT NULL                COMMENT "所属文章ID",
    `parent_id`   BIGINT   DEFAULT NULL            COMMENT "父评论ID（回复评论时用）",
    `content`     TEXT     NOT NULL                COMMENT "评论内容",
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_article_id` (`article_id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_parent_id` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT="评论表";

-- ========================================
-- 4. 点赞记录表
-- ========================================
CREATE TABLE `like_record` (
    `id`          BIGINT   NOT NULL AUTO_INCREMENT COMMENT "主键ID",
    `user_id`     BIGINT   NOT NULL                COMMENT "点赞用户ID",
    `article_id`  BIGINT   NOT NULL                COMMENT "被点赞文章ID",
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_article` (`user_id`, `article_id`),
    KEY `idx_article_id` (`article_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT="点赞记录表";
```

### 3.3 字段设计要点说明

| 字段 | 设计理由 |
|------|---------|
| `password` 长度 128 | BCrypt 输出固定 60 字符，留余量 |
| `like_count` 在 article 表冗余 | 避免每次 COUNT 点赞表，提升列表查询性能 |
| `comment_count` 在 article 表冗余 | 同上 |
| `uk_user_article` 唯一索引 | 天然防止重复点赞（幂等） |
| `parent_id` 可为 NULL | NULL 表示顶级评论，有值表示回复 |
| `tags` 用逗号分隔字符串 | 初期简化设计；后期可拆为 tag 关联表 |

### 第 5 步：JWT 工具类 + Spring Security 配置（最难但最关键的一步）

#### 5.1 JWT 工具类

```java
// utils/JwtUtil.java
@Component
public class JwtUtil {

    private static final String SECRET = "my-blog-secret-key-which-should-be-at-least-256-bits-long-for-hs256";
    private static final long EXPIRATION = 7 * 24 * 60 * 60 * 1000L; // 7 天

    public String generateToken(Long userId, String username) {
        return Jwts.builder()
            .claim("userId", userId)
            .claim("username", username)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + EXPIRATION))
            .signWith(getSecretKey())
            .compact();
    }

    public Claims parseToken(String token) {
        return Jwts.parser()
            .verifyWith(getSecretKey())
            .build()
            .parseSignedClaims(token)
            .getPayload();
    }

    public boolean isExpired(String token) {
        try {
            return parseToken(token).getExpiration().before(new Date());
        } catch (Exception e) {
            return true;
        }
    }

    private SecretKey getSecretKey() {
        return Keys.hmacShaKeyFor(SECRET.getBytes(StandardCharsets.UTF_8));
    }
}
```

#### 5.2 JWT 认证过滤器

```java
// config/JwtAuthenticationFilter.java
@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        String authHeader = request.getHeader("Authorization");
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        String token = authHeader.substring(7);
        try {
            Claims claims = jwtUtil.parseToken(token);
            Long userId = claims.get("userId", Long.class);
            String username = claims.get("username", String.class);

            UsernamePasswordAuthenticationToken authentication =
                new UsernamePasswordAuthenticationToken(
                    new LoginUser(userId, username), null, List.of());
            authentication.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
            SecurityContextHolder.getContext().setAuthentication(authentication);
        } catch (Exception e) {
            // Token 无效，不设置认证信息
        }

        filterChain.doFilter(request, response);
    }
}
```

#### 5.3 Spring Security 配置

```java
// config/SecurityConfig.java
@Configuration
@RequiredArgsConstructor
@EnableWebSecurity
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthenticationFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(
                    "/api/user/register",
                    "/api/user/login",
                    "/api/article/**",
                    "/api/comment/article/**",
                    "/doc.html",
                    "/swagger-ui/**",
                    "/v3/api-docs/**",
                    "/webjars/**"
                ).permitAll()
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

**新手最容易踩的坑**：
- CSRF 不关 → POST 请求 403 → 关掉
- 公共接口没放行 → 游客无法查看文章 → 加 `permitAll()`
- 过滤器顺序不对 → 用 `addFilterBefore`

#### 5.4 @CurrentUser 注解 + 解析器

```java
// common/CurrentUser.java
@Target(ElementType.PARAMETER)
@Retention(RetentionPolicy.RUNTIME)
```

```java
// config/CurrentUserResolver.java
@Component
public class CurrentUserResolver implements HandlerMethodArgumentResolver {
    @Override
    public boolean supportsParameter(MethodParameter parameter) {
        return parameter.hasParameterAnnotation(CurrentUser.class)
            && parameter.getParameterType().equals(LoginUser.class);
    }

    @Override
    public Object resolveArgument(MethodParameter parameter, ModelAndViewContainer container,
                                  NativeWebRequest webRequest, WebDataBinderFactory factory) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null && auth.getPrincipal() instanceof LoginUser loginUser) {
            return loginUser;
        }
        return null;
    }
}
```

```java
// config/WebMvcConfig.java

public class WebMvcConfig implements WebMvcConfigurer {
    private final CurrentUserResolver currentUserResolver;

    @Override
    public void addArgumentResolvers(List<HandlerMethodArgumentResolver> resolvers) {

    }
}
```

#### Service 实现

```java
// service/impl/UserServiceImpl.java
@Service
@RequiredArgsConstructor
public class UserServiceImpl implements UserService {

    private final UserMapper userMapper;
    private final PasswordEncoder passwordEncoder;
    private final JwtUtil jwtUtil;
    private final UserConverter userConverter;

    @Override
    @Transactional
    public UserVO register(RegisterDTO dto) {
        Long count = userMapper.selectCount(
            Wrappers.<User>lambdaQuery().eq(User::getUsername, dto.getUsername()));
        if (count > 0) {
            throw new BusinessException(400, "用户名已存在");
        }

        User user = userConverter.toEntity(dto);
        user.setPassword(passwordEncoder.encode(dto.getPassword()));
        user.setNickname(dto.getUsername());

        userMapper.insert(user);
        return userConverter.toVO(user);
    }

    @Override
    public LoginResponseDTO login(LoginDTO dto) {
        User user = userMapper.selectOne(
            Wrappers.<User>lambdaQuery().eq(User::getUsername, dto.getUsername()));
        if (user == null || !passwordEncoder.matches(dto.getPassword(), user.getPassword())) {
            throw new BusinessException(401, "用户名或密码错误");
        }

        String token = jwtUtil.generateToken(user.getId(), user.getUsername());
        LoginResponseDTO res = new LoginResponseDTO();
        res.setToken(token);
        res.setUser(userConverter.toVO(user));
        return res;
    }

    @Override
    public UserVO getUserById(Long id) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new BusinessException(404, "用户不存在");
        }
        return userConverter.toVO(user);
    }
```

```java
// service/impl/LikeServiceImpl.java
@Service
@RequiredArgsConstructor
public class LikeServiceImpl implements LikeService {

    private final LikeRecordMapper likeRecordMapper;
    private final ArticleMapper articleMapper;

    @Override
    @Transactional
    public void like(Long articleId, Long userId) {
        Article article = articleMapper.selectById(articleId);
        if (article == null) {
            throw new BusinessException(404, "文章不存在");
        }

        LikeRecord record = new LikeRecord();
        record.setUserId(userId);
        record.setArticleId(articleId);

        try {
            likeRecordMapper.insert(record);
            articleMapper.incrLikeCount(articleId);
        } catch (DuplicateKeyException e) {
            // UNIQUE KEY (user_id, article_id) 冲突 → 已点赞，幂等返回
            throw new BusinessException(400, "已经点过赞了");
        }
    }

    @Override
    @Transactional
    public void unlike(Long articleId, Long userId) {
        int deleted = likeRecordMapper.delete(
            Wrappers.<LikeRecord>lambdaQuery()
                .eq(LikeRecord::getUserId, userId)
                .eq(LikeRecord::getArticleId, articleId));
        if (deleted > 0) {
            articleMapper.decrLikeCount(articleId);
        }
    }
}
```

```java
// service/impl/CommentServiceImpl.java
@Service
@RequiredArgsConstructor
public class CommentServiceImpl implements CommentService {

    private final CommentMapper commentMapper;
    private final ArticleMapper articleMapper;
    private final UserMapper userMapper;
    private final CommentConverter commentConverter;

    @Override
    @Transactional
    public CommentVO create(CommentCreateDTO dto, Long userId) {
        Article article = articleMapper.selectById(dto.getArticleId());
        if (article == null) {
            throw new BusinessException(404, "文章不存在");
        }

        if (dto.getParentId() != null) {
            Comment parent = commentMapper.selectById(dto.getParentId());
            if (parent == null) {
                throw new BusinessException(404, "父评论不存在");
            }
        }

        Comment comment = commentConverter.toEntity(dto);
        comment.setUserId(userId);
        commentMapper.insert(comment);

        articleMapper.incrCommentCount(dto.getArticleId());

        CommentVO vo = commentConverter.toVO(comment);
        User user = userMapper.selectById(userId);
        if (user != null) {
            vo.setAuthorName(user.getNickname());
            vo.setAuthorAvatar(user.getAvatar());
        }
        return vo;
    }

    @Override
    public PageResult<CommentVO> getComments(Long articleId, int page, int size) {
        // 先查顶级评论（parent_id IS NULL）
        Page<Comment> commentPage = commentMapper.selectPage(
            new Page<>(page, size),
            Wrappers.<Comment>lambdaQuery()
                .eq(Comment::getArticleId, articleId)
                .isNull(Comment::getParentId)
                .orderByDesc(Comment::getCreateTime));

        List<CommentVO> list = commentConverter.toVOList(commentPage.getRecords());

        // 为每条顶级评论填充作者 + 子评论
        list.forEach(vo -> {
            fillAuthor(vo);
            // 查询该顶级评论下的回复
            List<Comment> replies = commentMapper.selectList(
                Wrappers.<Comment>lambdaQuery()
                    .eq(Comment::getParentId, vo.getId())
                    .orderByAsc(Comment::getCreateTime));
            List<CommentVO> replyVOs = commentConverter.toVOList(replies);
            replyVOs.forEach(this::fillAuthor);
            vo.setReplies(replyVOs);
        });

        return new PageResult<>(list, commentPage.getTotal(),
            commentPage.getCurrent(), commentPage.getSize());
    }

    private void fillAuthor(CommentVO vo) {
        User user = userMapper.selectById(vo.getUserId());
        if (user != null) {
            vo.setAuthorName(user.getNickname());
            vo.setAuthorAvatar(user.getAvatar());
        }
    }

    @Override
    @Transactional
    public void delete(Long id, Long userId) {
        Comment comment = commentMapper.selectById(id);
        if (comment == null) {
            throw new BusinessException(404, "评论不存在");
        }
        if (!comment.getUserId().equals(userId)) {
            throw new BusinessException(403, "无权删除他人评论");
        }

        // 同时删除该评论下的所有子评论
        commentMapper.delete(Wrappers.<Comment>lambdaQuery()
            .eq(Comment::getId, id)
            .or()
            .eq(Comment::getParentId, id));
        articleMapper.decrCommentCount(comment.getArticleId());
    }
}
```

### 各包职责

| 包 | 职责 |
|----|------|
| `config` | 框架配置类，组装各个组件 |
| `controller` | 接收 HTTP 请求，调用 Service，返回 Result |
| `service` / `impl` | 业务逻辑层，写具体规则 |
| `mapper` | 数据库操作，继承 BaseMapper 免写 SQL |
| `entity` | 与数据库表一一对应 |
| `dto` | 请求参数封装（前端→后端） |
| `vo` | 响应数据封装（后端→前端），不暴露敏感字段 |
| `common` | 工具性代码：异常、响应体、注解、转换器 |
| `utils` | 纯工具类（JWT 等） |

### 核心原则

1. **不准 Entity 直接返回前端** → 必须通过 VO
2. **不准密码明文存储** → 必须 BCrypt
3. **不准信任前端传的 userId** → 从 Token 中拿
4. **每个 Service 方法只做一件事** → 单一职责

## 7. 附录：向 AI 提问省 Token 的技巧

### 7.1 为什么 Token 用得快？

每次你发送的长问题 + AI 返回的长回答 = Token 消耗。以下技巧让你用**更少的 Token 得到更好的答案**。

### 7.2 精准描述（而非模糊指令）

| ❌ 浪费 Token | ✅ 节省 Token |
|---------------|---------------|
| "帮我写一个博客系统，要功能完整，界面好看" | "生成博客 UserController，含登录和注册接口，JWT 返回 token" |
| "我不太懂这个，能详细讲一下吗" | "@PostMapping 和 @RequestMapping 有什么区别？一句话回答" |

### 7.3 分段追问（而非一次性说完）

**错误做法**（一次消耗 5K Token）：
> "帮我从零搭建博客项目，包括数据库设计、后端代码、前端界面……"

**正确做法**（分段 → 每次 1K Token）：
```
第1轮：帮我设计博客数据库表，给出建表 SQL
第2轮：基于上一步，生成 Spring Boot application.yml
第3轮：生成 UserController 注册/登录代码
第4轮：生成 ArticleController CRUD
```

### 7.4 要求"只输出代码骨架"

```
请生成 ArticleController，只输出 Java 代码，不要文字解释。
```

→ 省去 AI 的啰嗦开场白和总结陈词。

### 7.5 指定格式，避免多余文字

```
用表格列出 8 个接口，每列：方法、路径、说明、需登录
```

→ 比用段落描述省 60% Token。

### 7.6 用缩写

```
JPA vs MyBatis 优劣势，各 3 点，每点 ≤10 字
```

### 7.7 先分析再输出

```
先分析下面代码的问题，再给出修正版。不要额外评论。
```

### 7.8 直接粘贴报错信息（不要自己描述）

```
这个报错是什么意思？
<粘贴完整错误日志>
```

→ AI 需要完整错误栈才能准确定位。

### 7.9 善用"继续"

当 AI 只生成了部分代码时：
```
继续，接着上面的 ArticleController 生成 ArticleServiceImpl
```

### 7.10 总结对照表

| 场景 | 省 Token 提问 |
|------|--------------|
| 要代码 | "生成 xxx，纯代码，无解释" |
| 要对比 | "表格对比 A 和 B，3 个维度" |
| 要排查 | "分析错误：<粘贴日志>" |
| 要补全 | "继续上一轮的 Service 实现" |
| 要精简 | "一句话回答" 或 "≤50 字" |
| 要方案 | "列出 3 种方案，每行：方案名、优缺点、适用场景" |
